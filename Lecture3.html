1. Introduction to Computer Architecture

The von Neumann Architecture

Computer Architecture: The design and organization of fundamental computer components and how they interact to process data.

Most modern computers follow the von Neumann architecture, which consists of four main components:

  • Central Processing Unit (CPU): The "brain" that executes instructions
  • Memory: Stores both data and instructions
  • Input/Output Devices: Interface with the external world
  • Bus System: Communication pathways between components

2. CPU Components & Functions

üî¢ Arithmetic/Logic Unit (ALU)

Performs all mathematical and logical operations:

  • Arithmetic: Addition, subtraction, multiplication, division
  • Logical: AND, OR, XOR, NOT operations
  • Bit Shifting: Shift and rotate operations

üéÆ Control Unit (CU)

Coordinates all CPU activities and manages data flow:

  • Directs operation of other CPU components
  • Manages fetch-decode-execute cycle
  • Controls data movement between CPU and memory

üíæ Registers

Small, fast storage locations inside the CPU:

  • General Purpose: Hold temporary data during processing
  • Special Purpose: Program Counter (PC), Instruction Register (IR)
  • Very Fast: Much faster than main memory access

3. Machine Language & Instruction Types

Understanding Machine Language

Machine Language: The lowest-level programming language consisting of binary codes (0s and 1s) that are directly executed by the CPU.

Each machine instruction typically has two parts:

  • Op-code: Specifies the operation to perform (e.g., 3 = STORE, 1 = LOAD)
  • Operand: Provides additional information about the operation (which register, memory address)
Instruction Type Purpose Examples
Data Transfer Move data between memory and registers LOAD, STORE, MOVE
Arithmetic/Logic Perform calculations and logical operations ADD, SUB, AND, OR, XOR
Control Change program flow and execution JUMP, BRANCH, CALL, HALT

4. Program Execution & Machine Cycle

The Fetch-Decode-Execute Cycle

Every instruction goes through three stages in a continuous cycle:

1. Fetch

The CPU retrieves the next instruction from memory:

  • Program Counter (PC) holds the address of next instruction
  • CPU sends address to memory via address bus
  • Instruction placed in Instruction Register (IR)

2. Decode

The CPU interprets the instruction:

  • Control Unit examines op-code in IR
  • Determines what operation to perform
  • Identifies which registers or memory addresses are involved

3. Execute

The CPU performs the actual operation:

  • ALU performs calculations if needed
  • Data moved between registers and memory
  • Results stored in appropriate locations

5. Logic, Shift, and Rotate Operations

Bit Manipulation Operations

These operations are fundamental to low-level programming:

Operation Symbol Description Example (Binary)
AND & Both bits must be 1 to get 1 1100 & 1010 = 1000
OR | At least one bit must be 1 to get 1 1100 | 1010 = 1110
XOR ^ Bits must be different to get 1 1100 ^ 1010 = 0110
Shift Left << Move bits left, fill with 0s 0011 << 1 = 0110
Shift Right >> Move bits right, fill with 0s 1100 >> 1 = 0110
